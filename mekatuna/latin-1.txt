<H1>ISO 8859-1 (Latin-1) Character Set</H1>

<P>
<STRONG>このページには、一文字ずつの細かい画像が大量に含まれています。そのため、Web ブラウザで「画像も表示する」設定にしている場合、表示終了まで時間がかかります。悪しからず。</STRONG>
</P>
<P>
ISO 8859-1 の Latin-1 (以降 Latin-1) は、アルファベットにウムラウトがついた文字など、英語以外のラテン系欧州語(フランス語、ドイツ語…)で使われるアルファベットを集めた文字セット (Character Set) です。<BR>
DOS/Windows, Mac の日本語環境では、文字コードが半角カタカナ(1バイトカタカナ)や日本語漢字と重なっているため、通常、日本語と同時に Latin-1 を入力表示することはできません。<BR>

Web ブラウザの場合は、HTML で Latin-1 を記述するための仕様が定められており、ブラウザが対応していれば、日本語と Latin-1 文字を同時に入力表示できることがあります。<BR>
たまたま日本語と一緒に Latin-1 文字を使いたいことがあったので、「Latin-1 を表示できるブラウザと設定の組みあわせは?」「できない組みあわせは?」を調べるために、実際に Latin-1 を記述したテーブルを作ってみました。
</P>

	<DT>結果

<H2>方法</H2>
<P>
HTML の仕様では、HTML 文書内に Latin-1 文字を記述する時、直接その文字を入力する以外に、Named Entity と Code Entity を使用する方法を提供しています。
(「<A HREF="http://www.w3.org/TR/REC-html32.html">HTML 3.2 Reference Specification</A>」中の "<A HREF="http://www.w3.org/TR/REC-html32.html#latin1">Character Entities for ISO Latin-1</A>")<BR>
例えば“ウムラウトつきの大文字 A”について、Named Entity は“&amp;Auml;”と名称で記述する方法、Code Entity は“&amp;#196;”と文字コードで記述する方法です。<BR>
この Named Entity と Code Entity を一覧にした<A HREF="#Table">テーブル</A>を作成しました。これをブラウザで表示し、Latin-1 文字が表示されるかどうかを見ます。

</P>
<P>
* Web ブラウザの「文字コード・セット」を切り換えることで、日本語以外の環境での見え方も検討することができます。
このために、本ページのヘッダから &lt;META HTTP-EQUIV="Content-Type" CONTENT="text/html; charset=iso-2022-jp"&gt; の明示的指定をとりのぞいています。これがあると、文字コードセットを切り換えてあっても、自動的に日本語に切り換わってしまいます。
</P>
<H3>更新記録</H3>
テーブルの &quot;Description&quot; を <A HREF="http://www.w3.org/TR/REC-html40/">HTML 4.0 Specification</A></I> (<A HREF="http://www.w3.org/">W3C - The World Wide Web Consortium</A>) の ISO 10646 (Unicode) 対応の記述にさしかえ。<SMALL>(Jun-1998)</SMALL>

<H2>結果</H2>

<H3>検証点</H3>
<UL>
	<LI><A HREF="#Table">テーブル</A>中の Named Entity による Character (1)、Code Entity による Character (2) それぞれの箇所に、Latin-1 文字が表示されるかどうかをみる。
	確認の目安は、画像と同じ文字が表示されるかどうか。<BR>
	画像の文字は、Windows NT 4.0 標準搭載のフォント Courier New (TrueType) 12pt を使用した。
</UL>

<H3>結論</H3>

<UL>
<LI>検証したブラウザの全部にわたり、日本語で確実に Latin-1 文字が表示されるものはない。Latin-1 文字を記述したければ、一部のブラウザで見られないことを意識しておく必要がある。
<LI>Latin-1 文字を記述する場合は、Named Entity を使用することが望ましい。Code Entity は、環境によって別の文字になる可能性が高い。
</UL>

<H3>検証したブラウザ</H3>
<DL>
<DT><STRONG>NC 4.04J: Windows 95 用 Netscape Communicator 4.04 (日本語版)</STRONG></DT>
	<DD>
	<B>手順：</B>メニューで [表示]-[文字コード セット]-[日本語(自動選択)] を選択して表示。<BR>

	<B>結果：</B>Named Entity, Code Entity とも一部の文字のみ表示可能。表示できない文字は <B>?</B> に置き換えられて表示される。
	</DD>

<DT><STRONG>NN2 2.02J: OS/2 Warp 用 Netscape Navigator 2.02 [ja]</STRONG></DT>
	<DD>
	<B>手順：</B>メニューで [オプション]-[文書の文字コード セット]-[日本語(自動判別)] を選択して表示。<BR>
	<B>結果：</B>Named Entity, Code Entity とも、対応する文字コードの1バイトカタカナに置きかえられてしまう。
	</DD>

<DT><STRONG>IE 3.02J: 日本語版 Windows 95 用 Microsoft インターネット エクスプローラ 3.02 (4.70.1300)</STRONG></DT>
	<DD>
	<B>手順：</B>右下の文字コードアイコンで [日本語(自動判別)] を選択して表示。<BR>
	<B>結果：</B>Named Entiry はすべて表示可能。Code Entity は、同じ文字コードを持った1バイトカタカナに置きかえられてしまう。
	</DD>
</DL>
